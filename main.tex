\section{Introduction}
\label{sec:introduction}

The current situation in video-language modeling is paradoxical. Video-LLMs can generate fluent descriptions and answer many semantic questions from RGB video \cite{zhang2023video, damonl2024videollama2, li2024llavanext}. However, when questions require stable 3D structure over time, such as tracking objects through occlusion, comparing motion under viewpoint changes, or maintaining correct temporal causality, performance degrades sharply. This gap is now quantified by spatiotemporal benchmarks, which reveal substantial deficits in temporal coherence, viewpoint-aware motion understanding, and physical plausibility \cite{zhou2025vlm4d, zhu20254dbench, wang2025spatial4dbench}.

These trends point to a central conclusion: the bottleneck is not a missing ``module'', but the backbone representation itself. This proposal targets the missing piece: a backbone that learns a unified 4D latent world representation from the most available modality, timestamped 2D video. 4D-MLLM is introduced as a 4D latent world model transformer designed around (1) changing the token definition from 2D patches to 4D entities, (2) modifying attention to be biased by spatiotemporal proximity, and (3) shifting the representation target from ``video features'' to a unified 4D latent world representation. These changes aim to bridge semantic video understanding and structured 3D plus time reasoning using ubiquitous 2D video inputs.

\section{Related Work}
\label{sec:related}

Early 4D MLLMs and 4D vision-language systems have begun to incorporate explicit spatiotemporal structure. Uni4D-LLM represents an early attempt to unify 4D understanding and generation by injecting spatiotemporal geometric cues into a shared Transformer representation \cite{zhou2025uni4dllm}. Other work demonstrates strong 4D modeling when explicit dynamic 3D inputs are available, such as dynamic point cloud sequences in 4DPC²hat \cite{4dpc2hat2026} or LiDAR sequences in B4DL \cite{b4dl2025}. More recently, frameworks like STLLM \cite{stllm2025} combine global 3D environment contexts with local egocentric video to bridge spatial layout and temporal actions.

Complementary directions include geometry-augmented video VLMs, such as DSR Suite \cite{dsrsuite2025}, which injects reconstruction-derived geometry tokens (poses, trajectories) via a specialized selection module. Structured scene representations also play a role, as seen in PSG-4D-LLM \cite{psg4dllm2025}, which ties LLM reasoning to 4D panoptic masks and temporal scene graphs. Robotics-focused efforts like 4D-VLA \cite{4dvla2025} and language-grounded dynamic fields like 4D LangSplat \cite{4dlangsplat2025} further emphasize the value of spatiotemporal grounding. Unlike these systems which often rely on explicit 3D sensors or reconstruction-heavy task heads, 4D-MLLM targets a general-purpose backbone paradigm that induces a structured 4D latent world directly from 2D video.

\section{Proposed Method}
\label{sec:methodology}

A backbone is proposed that transforms the fundamental unit of computation from a ``2D patch'' to a ``4D token''. Concretely, for each token a tuple $T_i = (f_i, x_i, \tau_i)$ is maintained, where $f_i$ is the visual feature (appearance), $x_i$ is a learned latent 3D coordinate that organizes tokens into a geometry-consistent space, and $\tau_i$ is the absolute timestamp associated with the observation. The key point is that $x_i$ is not ground-truth geometry and does not rely on explicit reconstruction; it is learned as part of the representation target so that the backbone naturally supports questions of ``what is where, when'' and can maintain object permanence across views and time.

To learn $x_i$ from timestamped 2D video, geometry is induced through self-supervised spatiotemporal consistency rather than SLAM-style pipelines. The model is trained to align tokens that correspond to the same physical entity across time and viewpoint changes, enforcing stability for static regions ($x_{i,t} \approx x_{j,t+1}$) while allowing structured evolution for dynamic regions. Time is modeled as a continuous variable using learnable Fourier features to handle variable framerates and precise durations. Importantly, the model learns absolute time understanding by encoding real-world timestamps directly in the representation, which enables consistent reasoning over long videos, irregular sampling, and multi-camera synchronization. Dynamics are represented with a time-conditioned predictor that relates changes in $x$ to elapsed time, encouraging temporal scaling consistency.

On top of these 4D tokens, attention is modified to reflect spatiotemporal structure: attention is biased so that tokens that are close in latent 3D space and close in time interact more strongly than distant ones, supporting locality and causal propagation in the learned world model. The backbone is trained with Masked 4D Modeling by masking a spatial region (in latent space) or a temporal interval and requiring reconstruction from the remaining context, which encourages both occlusion reasoning and trajectory completion. Evaluation treats 4D-MLLM as a backbone rather than a task-specific head, using benchmarks that expose 4D reasoning gaps including VLM4D \cite{zhou2025vlm4d}, 4D-Bench \cite{zhu20254dbench}, and Spatial4D-Bench \cite{wang2025spatial4dbench}, with comparisons to strong video-language baselines \cite{damonl2024videollama2, li2024llavanext} and ablations over token definition, attention bias, and representation objectives.

\section{Expected Outcomes}
\label{sec:outcomes}

4D-MLLM is expected to deliver a backbone that moves beyond sequence modeling and produces a stable, evolving 4D latent world representation that is useful across downstream tasks. In practice, this should translate into stronger 3D-aware question answering (e.g., resolving relative depth and spatial relations under viewpoint changes), more reliable metrology-style reasoning (e.g., inferring speed and trajectory from changes in latent position over absolute time), and improved occlusion reasoning through object permanence, where entities remain represented even when temporarily invisible in the raw video stream.
