\begin{abstract}
Current Video-LLMs treat time merely as a frame index and space as flattened 2D patches, lacking explicit 3D modeling and continuous temporal reasoning. This limitation prevents them from truly "understanding" the physical dynamics of a scene, such as quantifying speed, trajectory, or causal relationships in 4D space. We propose **ST-MLLM** (Structured SpatioTemporal Multimodal Large Model), a novel framework that induces a structured 4D latent representation directly from 2D video and timestamps. Instead of heavy explicit 3D reconstruction (e.g., NeRF), ST-MLLM utilizes a **Geometry Induction Module** to lift 2D tokens into motion-aware 3D coordinates and a **Continuous Time Encoding** to model temporal causality. By conditioning a Large Language Model on these structured 4D tokens via a SpatioTemporal Transformer, we aim to bridge the gap between semantic video description and precise physical metrology.
\end{abstract}

\section{Introduction}
\label{sec:introduction}

Standard Video Multimodal Large Language Models (Video-MLLMs) suffer from a fundamental disconnect between their input representation and the physical world. They process video as a sequence of 2D images, where "time" is reduced to a discrete sequence index and "space" is compressed into feature vectors. While effective for semantic captioning, this approach fails at **Metrology**—the precise quantification of physical dynamics. They cannot reliably answer "How fast is the car moving?" or "Will these two objects collide?" because they lack an internal model of 3D geometry and continuous time.

Conversely, "Native 4D" approaches that rely on explicit inputs like LiDAR or full 3D point cloud sequences \cite{4dpc2hat2026, b4dl2025} are computationally expensive and limited by data availability. We argue for a middle path: **inducing** 4D structure from ubiquitous 2D video.

We introduce **ST-MLLM**, a framework designed to construct a 3D latent structure and model continuous time explicitly. By lifting 2D visual tokens into a 3D-aware latent space and fusing them with continuous timestamp embeddings, ST-MLLM learns to reason about spatial proximity and temporal causality, enabling capabilities previously reserved for specialized physics engines.

\section{Methodology: ST-MLLM Framework}
\label{sec:methodology}

Our model processes a video sequence $\mathcal{V} = \{(I_t, \tau_t)\}_{t=1}^{T}$, consisting of RGB frames $I_t$ and real-world timestamps $\tau_t$. The architecture consists of five core stages:

\textbf{1. Visual Encoder.} We extract standard 2D visual features using a pre-trained backbone (e.g., CLIP-ViT or SigLIP), yielding a sequence of 2D patch tokens.

\textbf{2. Geometry Induction Module (The Core).} Instead of explicit reconstruction, we learn a mapping $Z_t = f_{geo}(I_t, I_{t-1})$ to produce motion-aware geometry tokens.
\begin{itemize}
    \item \textit{Pseudo-Depth & Optical Flow:} We utilize lightweight or frozen heads to estimate monocular depth maps and frame-to-frame optical flow.
    \item \textit{2D-to-3D Lifting:} We lift each 2D patch token $p_i$ with coordinates $(u_i, v_i)$ and estimated depth $d_i$ into a latent 3D coordinate space: $x_i = (u_i, v_i, d_i) \rightarrow \mathbb{R}^3$. This transforms flat visual tokens into geometry-aware tokens $G_t$.
\end{itemize}

\textbf{3. Temporal Structure Module.} To move beyond discrete frame indices, we employ **Continuous Time Encoding** $\gamma(\tau_t)$. This module combines learnable time embeddings with sinusoidal encodings of real-world timestamps (e.g., hours, seconds), allowing the model to understand irregular sampling rates and precise durations.

\textbf{4. 4D Token Mixer: SpatioTemporal Transformer.} We fuse the visual, geometric, and temporal tokens using a specialized Transformer. Crucially, we introduce a **3D-Aware Attention Bias**:
\[
A_{ij} = \text{Attention}(Q_i, K_j) + f_{\text{bias}}(|x_i - x_j|_{3D}, |\tau_i - \tau_j|)
\]
This bias modulates attention weights based on 3D spatial distance and temporal interval, explicitly encouraging the model to attend to tokens that are physically close in space-time or causally related, rather than just semantically similar.

\textbf{5. Training Objectives.} We employ three physics-informed losses:
\begin{itemize}
    \item \textit{SpatioTemporal Contrastive Loss:} Forces the model to distinguish between (Same Location, Different Time) and (Different Location, Same Time) pairs.
    \item \textit{Future State Prediction:} $Z_{t+1} = f(Z_t, \text{motion})$, requiring the model to predict the next latent state based on current physics.
    \item \textit{3D Consistency Constraint:} Enforces that static objects maintain consistent 3D latent representations across time $f(x, t_1) \approx f(x, t_2)$.
\end{itemize}

\section{Expected Outcomes}
\label{sec:outcomes}

We expect ST-MLLM to demonstrate superior performance in:
\begin{itemize}
    \item \textbf{3D-Aware QA:} Answering questions about relative depth and spatial relationships without explicit 3D inputs.
    \item \textbf{Precise Metrology:} Estimating speed and trajectory by reasoning over continuous time and induced 3D structure.
    \item \textbf{Causal Reasoning:} Understanding event sequences based on physical proximity rather than just visual co-occurrence.
\end{itemize}

\section{Related Work}
\label{sec:related}

This work bridges the gap between **Video-LLMs** \cite{zhang2023video, ren2023timechat}, which lack 3D structure, and **Native 4D Models** \cite{4dpc2hat2026, b4dl2025}, which require heavy 3D inputs. While recent work like DSR Suite \cite{dsrsuite2025} augments VLMs with geometry tokens, our Geometry Induction Module offers a more end-to-end approach, internalizing the 2D-to-3D lifting process directly within the model's latent space.
